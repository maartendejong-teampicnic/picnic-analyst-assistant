# Skill: Write Confluence Pages

## What this covers
Writing and updating Confluence pages: analysis write-ups, A/B test documentation,
project specs, weekly metrics, and how-to guides.
Pages must be structured, conclusion-first, and use Confluence-native formatting elements.

## When to use
When creating new Confluence pages or updating existing ones — analysis write-ups, project documentation, weekly reports.

## How to do it

1. **Read the existing page** (if updating) via Confluence MCP before making any changes
2. **Identify page type**: analysis write-up / A/B result / project spec / how-to / weekly report
3. **Confirm target space** with Maarten before writing (ask if unknown)
4. **Draft** in the standard structure below — title as conclusion, TL;DR first
5. **Apply status macro**: Draft while writing, In Review when ready for feedback, Final when done
6. **Show draft** to Maarten before publishing/updating with `STATUS: NEEDS_APPROVAL`

### Execution tools

#### Read a page
Use Confluence MCP (`mcp-atlassian`):
```
Base URL: https://picnic.atlassian.net
```

#### Write / update a page
Use Confluence MCP to push content. Confirm space and parent page with Maarten first.

---

## Reference: Conventions & Patterns

### Language & format
- **Always English** — no exceptions for Confluence
- Use Confluence-native elements: status macros, tables, info panels, page properties
- Read before write: always read the existing page before making any update
